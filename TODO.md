## 결정 기록

- 2026-07-15: 온라인 DB는 MySQL 계열·InnoDB·utf8mb4를 기준으로 작성하고 사용자가 닷홈 DB에 최소 스키마를 적용했다. 실제 PHP 통합 동작과 서버 DB 버전은 배포 단계에서 별도로 검증한다.
- 2026-07-15: 초기 DB 사용량과 운영 복잡도를 줄이기 위해 온라인 MVP 스키마를 `gunnam_account`, `gunnam_post` 두 테이블로 축소한다. 카드와 답변은 `post_type`과 `parent_id`로 구분하며, 직접 건넴 데이터는 DB에 저장하지 않는다.
- 2026-07-15: P2 링크 카드는 관리 토큰을 URL fragment로 한 번만 전달하고 HttpOnly 세션으로 교환한다. 비회원 생성 API에는 파일 기반 요청 제한을 적용하며, 답변함은 HMAC 서명 cursor로 조회한다.
- 2026-07-16: 숫자 공개는 모든 유효 답변의 익명 집계를 포함하고, 전체 공개는 제출자가 명시적으로 동의한 선택·닉네임·추가 의견만 노출한다. 기존 비동의 답변은 설정 변경 후에도 공개하지 않는다.
- 2026-07-15: P1에서 `/create/mode`를 추가하되 기존 직접 건넴의 `cardData`와 localStorage 계약은 변경하지 않는다. 링크·모두에게 건넴은 유효 경로가 함께 설정될 때만 활성화되는 기본 비활성 기능 플래그로 둔다.
- 2026-07-13: 질문 생성이 아닌 유효한 답변 제출을 히스토리 생성 기준으로 삼는다. 히스토리는 카드 데이터와 분리 저장하고, 받은 답변을 중심으로 표시하며 질문은 펼칠 수 있는 보조 맥락으로 제공한다.
- 2026-07-13: 홈 진입 동작을 `새 카드 만들기`와 `기존 카드 만들기`로 분리한다. 새 카드는 localStorage를 초기값으로 복구하고, 기존 카드는 저장된 작성 데이터를 유지한다.
- 2026-07-13: 사용자 요청에 따라 단일 세션 범위를 확장해 원본 `cardData`를 `gunnam.cardData.v1` 키로 자동 저장·복구한다.
- 2026-07-10: 현재 저장소는 문서만 존재하므로 코드 관련 항목은 모두 미완료 상태에서 시작한다.
- 2026-07-10: 개발 순서는 기반 → 상태·라우팅 → 공통 UI → 작성자 흐름 → 응답자 흐름 → 품질 검증으로 정한다.
