# 테스트 체크리스트

`npm run dev`로 개발 서버를 실행한 상태에서 브라우저로 수동 테스트한다.

## 기본 실행

- [ ] Vite 개발 서버가 정상 실행된다.
- [ ] 첫 화면이 모바일 우선 레이아웃으로 표시된다.
- [ ] 데스크톱 폭에서 앱 화면이 중앙 모바일 프레임처럼 보인다.

## HomePage

- [ ] `건넴` 서비스명이 표시된다.
- [ ] `카드 만들기` 버튼으로 CreateCardPage로 이동한다.
- [ ] 로그인 없이 바로 사용할 수 있다는 안내가 보인다.

## CreateCardPage

- [ ] 메인 카드 단계에서 카드 안에 직접 문구를 입력할 수 있다.
- [ ] 메인 카드 이미지를 선택하고 미리볼 수 있다.
- [ ] 메인 카드 이미지가 잘리지 않고 원본 비율로 표시된다.
- [ ] 메인 카드 이미지가 없으면 이미지 영역이 남지 않는다.
- [ ] 수락 버튼 문구를 입력할 수 있다.
- [ ] 거절 버튼 문구를 입력할 수 있다.
- [ ] 수락 후 이미지/QR을 선택하고 미리볼 수 있다.
- [ ] 거절 후 이미지를 선택하고 미리볼 수 있다.
- [ ] 이미지를 다시 선택할 수 있다.
- [ ] 이미지를 삭제할 수 있다.
- [ ] 수락 후 메시지를 입력할 수 있다.
- [ ] 거절 후 메시지를 입력할 수 있다.
- [ ] 수락 후 답장 입력 ON/OFF를 바꿀 수 있다.
- [ ] 메인 카드 내용이 비어 있으면 미리보기로 이동하지 않는다.
- [ ] 메인 카드 내용이 비어 있으면 `상대에게 보여줄 내용을 적어주세요.`가 표시된다.

## CreateCardPage 단계 흐름

- [ ] 1단계 메인 카드에서 2단계 수락 페이지로 이동할 수 있다.
- [ ] 2단계 수락 페이지에서 3단계 거절 페이지로 이동할 수 있다.
- [ ] 이전 버튼으로 전 단계로 돌아갈 수 있다.
- [ ] 수락 페이지 단계에서 수락 결과 화면 규칙이 적용된다.
- [ ] 거절 페이지 단계에서 거절 결과 화면 규칙이 적용된다.
- [ ] 이미지가 없으면 이미지 영역이 남지 않는다.
- [ ] 수락 후 메시지가 비어 있으면 수락 버튼 문구가 표시된다.
- [ ] 거절 후 메시지가 비어 있으면 거절 버튼 문구가 표시된다.
- [ ] 답장 입력 ON이면 수락 페이지 단계에 답장 입력 영역이 보인다.
- [ ] `상대에게 보여주기` 버튼으로 ShowCardPage로 이동한다.

## ShowCardPage

- [ ] 메인 카드 문구가 중앙에 크게 표시된다.
- [ ] 수락 버튼이 표시된다.
- [ ] 거절 버튼이 표시된다.
- [ ] 수정, 홈, 뒤로가기, 설정, 공유, 설명 문구가 표시되지 않는다.
- [ ] 수락 버튼을 누르면 AcceptResultPage로 이동한다.
- [ ] 거절 버튼을 누르면 RejectResultPage로 이동한다.

## AcceptResultPage

- [ ] 수락 후 이미지/QR이 있으면 표시된다.
- [ ] 수락 후 이미지/QR이 없으면 이미지 영역이 없다.
- [ ] 수락 후 메시지가 있으면 해당 메시지가 표시된다.
- [ ] 수락 후 메시지가 없으면 수락 버튼 문구가 크게 표시된다.
- [ ] 답장 입력 OFF이면 답장 입력창이 없다.
- [ ] 답장 입력 ON이면 답장 입력창이 보인다.
- [ ] 빈 답장을 제출하면 `메시지를 입력해주세요.`가 표시된다.
- [ ] 답장을 입력하고 제출하면 ReplyCard가 표시된다.

## RejectResultPage

- [ ] 거절 후 이미지가 있으면 표시된다.
- [ ] 거절 후 이미지가 없으면 이미지 영역이 없다.
- [ ] 거절 후 메시지가 있으면 해당 메시지가 표시된다.
- [ ] 거절 후 메시지가 없으면 거절 버튼 문구가 크게 표시된다.
- [ ] 거절 사유 입력, 답장 입력, 다시 묻기 버튼이 없다.

## 반응형 확인

- [ ] 320px 폭에서도 주요 텍스트와 버튼이 깨지지 않는다.
- [ ] 390px 폭에서 모바일 앱처럼 자연스럽다.
- [ ] 768px 이상에서 중앙 정렬된 모바일 프레임처럼 보인다.
- [ ] 버튼 텍스트가 길어져도 레이아웃이 무너지지 않는다.
