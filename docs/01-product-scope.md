# 제품 범위와 사용자 흐름

## 제품 목적

건넴은 작성자가 말하기 어려운 질문이나 제안을 카드로 구성하고, 같은 기기에서 상대가 수락 또는 거절로 응답하게 하는 모바일 중심 웹 MVP다.

## 핵심 사용자 흐름

```text
HomePage (/)
  → CreateCardPage (/create)
      → 1단계 메인 카드
      → 2단계 수락 페이지
      → 3단계 거절 페이지
  → ShowCardPage (/show)
      ├─ 수락 → AcceptResultPage (/result/accept)
      └─ 거절 → RejectResultPage (/result/reject)
  → HistoryPage (/history)
```

- 별도 PreviewPage를 만들지 않는다.
- CreateCardPage의 각 단계가 편집 화면과 미리보기를 겸한다.
- 알 수 없는 URL은 홈으로 이동한다.

## MVP 필수 범위

- 새 카드 초기화 시작과 기존 로컬 작성 상태 복구
- 메인 문구 및 선택적 메인 이미지
- 수락·거절 버튼 문구
- 수락·거절 결과 문구 및 선택적 이미지
- 결과별 독립적인 답장 ON/OFF
- 공백 답장 검증 및 제출된 답장 카드 표시
- 버튼과 결과 문구 fallback
- 이미지 조건부 렌더링
- localStorage를 이용한 작성 데이터 자동 저장과 새로고침 복구
- 제출된 답변 중심의 히스토리 저장, 필터, 맥락 펼치기와 개별 삭제
- 최대 430px 모바일 프레임과 420px 이하 전체 화면 대응

## 명시적 범위 밖

- 회원가입, 로그인, 계정 및 권한
- 서버, API, 데이터베이스
- 링크 생성, 공유, 다른 기기 전달
- 이미지 서버 업로드
- 위치 정보, 실시간 채팅, 알림, 통계
- 거절 사유 분류, 다시 묻기

## 화면별 최소 요구사항

### HomePage

- 서비스명, 소개, 로그인 없이 사용 안내, 새 카드·기존 카드·카드 히스토리 버튼
- 새 카드는 로컬 상태 초기화, 기존 카드는 로컬 상태 유지

### CreateCardPage

- 자유롭게 이동 가능한 3개 단계 탭
- 첫 단계의 이전은 홈, 나머지는 직전 단계로 이동
- 중간 이동에는 필수 검증을 적용하지 않음
- 마지막 보여주기에서만 `mainText.trim()` 검증

### ShowCardPage

- 메인 카드와 수락·거절 버튼만 핵심 UI로 표시
- 수락 및 거절 결과 경로로 이동
- 뒤로가기는 작성 화면으로 이동

### 결과 페이지

- 해당 결과 이미지와 fallback이 적용된 결과 문구
- 해당 답장 토글이 켜진 경우에만 답장 입력 표시
- 제출된 답장을 입력 폼 위 카드로 표시
- 뒤로가기는 ShowCardPage로 이동

### HistoryPage

- 유효한 답변이 실제 제출된 기록만 표시
- 받은 답변을 중심 정보로, 질문은 펼칠 수 있는 보조 맥락으로 표시
- 전체·수락·거절 필터와 확인 후 개별 삭제 제공
