# UI 가이드

## 핵심 원칙

- 상대가 1초 안에 이해할 수 있어야 한다.
- 버튼은 크고 명확해야 한다.
- 카드 문구는 화면 중앙에 크게 배치한다.
- 이미지가 없으면 빈 공간을 남기지 않는다.
- 메시지가 없으면 버튼 문구를 결과 메시지로 사용한다.
- 수락한 경우에만 답장 입력을 허용한다.
- 거절은 빠르고 깔끔하게 끝낸다.

## 레이아웃

- 모바일 우선으로 설계한다.
- 기본 화면 폭은 360px 안팎을 기준으로 잡고, 좁은 화면에서는 좌우 여백을 유지한다.
- 데스크톱에서는 앱 화면이 중앙에 놓인 모바일 프레임처럼 보이게 한다.
- 카드 안의 텍스트는 너무 많은 설명을 붙이지 않고 핵심 문구만 크게 보여준다.

## 색과 분위기

레퍼런스 이미지 없이 진행하므로 과한 장식보다 조용하고 명확한 구성을 우선한다.

- 배경: 밝고 차분한 색
- 카드: 배경과 구분되는 흰색 또는 아주 옅은 색
- 수락 버튼: 긍정적이지만 과하게 튀지 않는 색
- 거절 버튼: 중립적이거나 낮은 대비의 색
- 에러 메시지: 짧고 분명하게 인지되는 색

## 버튼

- 수락/거절 버튼은 터치하기 쉬운 높이와 너비를 가진다.
- ShowCardPage에서는 수락/거절 버튼 외 보조 액션을 두지 않는다.
- 버튼 문구가 길어져도 줄바꿈 또는 안정적인 폭으로 깨지지 않게 한다.

## 입력 화면

- CreateCardPage는 작업 화면이므로 필요한 라벨과 설명을 제공한다.
- 한 화면이 너무 길어질 수 있으므로 섹션을 명확히 나눈다.
- 하단 `미리보기` 버튼은 쉽게 찾을 수 있어야 한다.

## 결과 화면

- 결과 메시지는 크게 표시한다.
- 이미지 또는 QR이 있을 때만 이미지 영역을 보여준다.
- 거절 결과는 입력 없이 짧게 끝낸다.

## 애니메이션

MVP 3순위 항목이다.

- 카드 전환에 짧은 fade 또는 slide를 사용할 수 있다.
- 버튼 클릭 피드백은 즉각적이어야 한다.
- 애니메이션이 기능 이해를 방해하면 제거한다.
