# 화면 흐름 및 페이지 요구사항

## 전체 흐름

```text
HomePage
→ CreateCardPage
→ ShowCardPage
→ AcceptResultPage 또는 RejectResultPage
```

수락 또는 거절 후 답장 입력이 ON인 경우:

```text
AcceptResultPage 또는 RejectResultPage
→ ReplyInput
→ ReplyCard
```

## HomePage

목적: 서비스 소개와 시작 진입.

표시 요소:

- 서비스명 `건넴`
- 짧은 설명 문구
- `카드 만들기` 버튼
- `로그인 없이 바로 사용` 안내

동작:

- `카드 만들기`: 빈 cardData로 CreateCardPage 이동

## CreateCardPage

목적: 상대에게 보일 화면을 단계별로 직접 작성.

단계:

- 1단계 메인 카드: 메인 카드 이미지, 메인 카드 문구, 수락 버튼 문구, 거절 버튼 문구
- 2단계 수락 페이지: 수락 후 이미지/QR, 수락 후 메시지, 수락 후 답장 입력 받기 ON/OFF
- 3단계 거절 페이지: 거절 후 이미지, 거절 후 메시지

동작:

- 각 단계는 실제 상대에게 보일 화면을 미리보기처럼 보여준다.
- 메인 카드 문구는 카드 안에서 직접 입력한다.
- 메인 이미지가 있으면 원본 비율을 유지해 잘리지 않게 표시한다.
- 이미지가 없으면 이미지 영역을 렌더링하지 않는다.
- `mainText`가 비어 있으면 다음 단계 또는 ShowCardPage로 이동하지 않고 `상대에게 보여줄 내용을 적어주세요.` 에러를 표시한다.
- 마지막 단계 하단에 `상대에게 보여주기` 버튼을 둔다.

## ShowCardPage

목적: 실제로 상대에게 폰을 보여주는 화면.

표시 요소:

- 메인 카드 문구
- 수락 버튼
- 거절 버튼

표시하지 말아야 할 요소:

- 수정 버튼
- 홈 버튼
- 뒤로가기 버튼
- 설정 버튼
- 공유 버튼
- 설명 문구
- 기타 불필요한 UI

동작:

- 수락 버튼: AcceptResultPage 이동
- 거절 버튼: RejectResultPage 이동

## AcceptResultPage

목적: 상대가 수락했을 때 보여주는 화면.

표시 요소:

- 수락 후 이미지/QR: 있으면 표시, 없으면 영역 제거
- 수락 결과 메시지: 수락 페이지 카드 안에서 직접 작성한 문구 표시. 비어 있으면 수락 버튼 문구 표시
- 답장 입력: `acceptReplyEnabled`가 true일 때만 표시. 기본값은 ON

답장 입력 규칙:

- 빈 답장은 제출할 수 없다.
- 빈 답장 제출 시 `메시지를 입력해주세요.` 에러를 표시한다.
- 답장을 제출하면 ReplyCard를 표시한다.
- 답장은 DB에 저장하지 않고 현재 화면에서만 보여준다.

## RejectResultPage

목적: 상대가 거절했을 때 보여주는 화면.

표시 요소:

- 거절 후 이미지: 있으면 표시, 없으면 영역 제거
- 거절 결과 메시지: 거절 페이지 카드 안에서 직접 작성한 문구 표시. 비어 있으면 거절 버튼 문구 표시
- 답장 입력: `rejectReplyEnabled`가 true일 때만 표시. 기본값은 ON

넣지 말아야 할 요소:

- 거절 사유 입력
- 다시 묻기 버튼
- 이유 선택 버튼
- 거절 통계 저장
