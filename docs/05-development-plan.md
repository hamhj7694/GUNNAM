# 개발 계획

진행 상태의 단일 기준은 루트 `TODO.md`다. 이 문서는 단계와 의존 관계를 설명하며, 실제 완료 여부는 `TODO.md`에서 갱신한다.

## 1단계: 프로젝트 기반

- Vite React 프로젝트와 npm 스크립트를 구성한다.
- 진입점과 전역 CSS를 연결한다.
- 권장 폴더 구조를 만든다.
- 최소 앱 빌드를 통과시킨다.

완료 결과: 빈 앱이 실행되고 프로덕션 빌드가 성공한다.

## 2단계: 데이터와 라우팅

- `initialCardData`를 정의한다.
- 부분 병합 업데이트를 구현한다.
- localStorage 자동 저장과 복구를 구현한다.
- fallback 계산 유틸을 구현한다.
- 5개 페이지 경로와 wildcard redirect를 구성한다.

완료 결과: 모든 경로가 렌더링되고 페이지 이동과 새로고침 후에도 카드 상태가 유지된다.

## 3단계: 공통 UI 기반

- 모바일 프레임과 기본 디자인 토큰을 구현한다.
- 카드, 버튼, 이미지, 결과 문구, 답장, 토글 컴포넌트를 구현한다.
- 조건부 이미지와 접근성 기본값을 적용한다.

완료 결과: 페이지에서 조합 가능한 공통 컴포넌트가 준비된다.

## 4단계: 작성자 흐름

- HomePage에서 저장 상태를 유지하는 시작 동작을 구현한다.
- CreateCardPage의 3단계 탭과 이전·다음 동작을 구현한다.
- 각 단계의 편집형 미리보기와 상태 연결을 구현한다.
- 최종 단계에서만 메인 문구를 검증한다.

완료 결과: 작성자가 PRD의 모든 카드 필드를 편집하고 상대 화면으로 이동할 수 있다.

## 5단계: 응답자 흐름

- ShowCardPage를 구현한다.
- 수락 및 거절 결과 화면을 구현한다.
- 결과별 답장 ON/OFF, 공백 검증, 답장 카드를 구현한다.

완료 결과: 수락과 거절의 전체 분기 및 답장 흐름이 동작한다.

## 6단계: 품질과 인수 검증

- fallback, 조건부 이미지, 상태 독립성을 회귀 검증한다.
- 320px, 390px, 데스크톱에서 반응형을 검증한다.
- 키보드, focus, aria label과 한글 인코딩을 확인한다.
- 프로덕션 빌드와 전체 수동 시나리오를 완료한다.

완료 결과: `PRD.md` 11절의 모든 인수 조건과 `docs/04-testing-and-done.md`를 충족한다.
