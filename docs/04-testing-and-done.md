# 테스트 규칙과 완료 정의

## 공통 완료 정의

작업은 다음 조건을 모두 만족할 때만 완료다.

- PRD의 해당 요구사항을 구현했다.
- 관련 화면과 반대 분기까지 회귀 확인했다.
- 빌드가 성공한다.
- 사용자 노출 한글이 깨지지 않는다.
- 320px, 390px, 데스크톱 프레임에서 레이아웃을 확인했다.
- 접근성 이름, 버튼 type, 키보드 focus를 확인했다.
- 관련 문서와 `TODO.md`를 최신화했다.

환경 문제로 검증하지 못한 경우 구현 완료와 검증 완료를 구분하며, 검증 항목을 미완료로 남긴다.

## 자동 검증

```text
npm install
npm run build
```

테스트 도구가 추가되기 전까지 빌드와 수동 시나리오를 최소 검증으로 사용한다. 테스트 도구 추가는 PRD 범위를 바꾸지 않는 범위에서 별도 결정한다.

## 핵심 수동 시나리오

### 시작과 라우팅

- 홈에서 새 카드 만들기를 누르면 저장 상태가 초기화되고 `/create`로 이동한다.
- 홈에서 기존 카드 만들기를 누르면 저장 상태를 유지하고 `/create`로 이동한다.
- 알 수 없는 경로는 홈으로 이동한다.
- 작성·표시·결과 화면의 뒤로가기가 지정 경로로 이동한다.

### 작성 흐름

- 세 단계 탭을 자유롭게 이동할 수 있다.
- 이전·다음 버튼이 경계에서 올바르게 동작한다.
- 빈 메인 문구로 최종 보여주기를 누르면 첫 단계 오류가 표시된다.
- 유효한 메인 문구는 `/show`로 이동한다.

### fallback

- 버튼 문구가 비었거나 공백뿐이면 `좋아요`, `싫어요`가 표시된다.
- 결과 문구가 비었으면 최종 버튼 문구가 표시된다.
- 직접 입력한 결과 문구가 있으면 fallback보다 우선한다.

### 이미지

- 메인·수락·거절 이미지를 각각 선택, 재선택, 삭제할 수 있다.
- 선택한 이미지는 해당 미리보기와 실제 화면에만 표시된다.
- 이미지가 없거나 삭제되면 빈 영역이 남지 않는다.
- 이미지가 잘리지 않고 비율을 유지한다.

### 결과와 답장

- 수락과 거절 버튼이 각각 올바른 결과 화면으로 이동한다.
- 각 답장 토글은 독립적으로 동작한다.
- 토글 OFF 결과 화면에는 답장 폼이 없다.
- 공백 답장은 오류가 표시되고 제출되지 않는다.
- 유효한 답장은 trim되어 저장되고 입력 폼 위 카드로 표시된다.
- 수락 답변 초기화는 수락 답변만 지우고, 거절 답변 초기화는 거절 답변만 지운다.
- 답변 초기화 후 답장 카드와 입력값이 사라지고 카드 본문과 결과 설정은 유지된다.
- 답변을 제출한 경우에만 히스토리 기록이 생성된다.
- 답변 수정은 기존 기록을 갱신하고, 초기화 후 새 답변은 새 기록을 생성한다.
- 히스토리 필터, 맥락 펼치기와 확인 후 개별 삭제가 동작한다.
- 새 카드 초기화 후에도 받은 답변 히스토리가 유지된다.

### 반응형과 접근성

- 320px에서 가로 스크롤과 겹침이 없다.
- 390px에서 모바일 전체 화면이 자연스럽다.
- 데스크톱에서 최대 430px 중앙 프레임이 보인다.
- 키보드로 탭, 버튼, 입력, 토글, 이미지 선택에 접근할 수 있다.
