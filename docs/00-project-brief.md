# 건넴 MVP 작업 브리프

## 제품 한 줄 설명

건넴은 말로 하기 애매한 질문, 부탁, 제안을 작은 카드로 보여주고, 상대가 수락 또는 거절 버튼으로 가볍게 응답할 수 있는 모바일 우선 웹앱이다.

## MVP 목표

- 사용자가 상대에게 보여줄 카드 문구와 수락/거절 후 결과 내용을 작성한다.
- 작성한 카드를 미리 확인한 뒤 실제로 상대에게 보여줄 수 있다.
- 상대는 수락 또는 거절만 선택한다.
- 수락 시 선택적으로 답장을 남길 수 있고, 답장은 현재 화면에서 카드 형태로만 표시한다.
- 모든 데이터는 React state로만 관리하며 새로고침 시 사라져도 된다.

## 기술 스택

- React
- Vite
- JavaScript
- CSS
- 모바일 우선 반응형 웹

## 구현하지 않는 것

- DB
- 로그인, 회원가입
- 서버 저장
- 위치 정보
- 주변 사람 탐색
- 채팅
- 통계
- 푸시 알림
- 거절 사유 수집
- 공유 기능

## 디자인 방향

레퍼런스 이미지 없이 진행한다.

- 심플한 모바일 카드 UI
- 큰 글씨와 넓은 여백
- 명확한 수락/거절 버튼
- 부담스럽지 않고 조용한 분위기
- 데스크톱에서는 모바일 화면이 중앙에 카드처럼 보이도록 처리
- ShowCardPage에는 상대가 눌러야 할 요소 외 불필요한 UI를 두지 않는다.

## 완료 기준 요약

- 메인 카드 문구를 작성할 수 있다.
- 메인 카드 문구가 비어 있으면 미리보기로 이동할 수 없다.
- 수락/거절 버튼 문구를 설정할 수 있다.
- 버튼 문구가 비어 있으면 기본값 `좋아요`, `싫어요`가 표시된다.
- 결과 메시지가 비어 있으면 해당 버튼 문구가 결과 메시지로 크게 표시된다.
- 이미지가 없으면 이미지 영역 자체를 렌더링하지 않는다.
- 수락 후 답장 입력 ON일 때만 답장 입력창이 표시된다.
- 답장 제출 후 ReplyCard가 표시된다.
- 거절 결과 화면에는 입력 기능이 없다.
- 모바일 화면에서 자연스럽게 사용할 수 있다.
