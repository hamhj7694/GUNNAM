# 건넴 디자인 에셋 체크리스트

포트폴리오 공개 및 실제 사용자 배포를 위한 에셋 목록이다. 먼저 최소 필수 세트를 준비하고, 포트폴리오와 PWA 에셋은 필요할 때 추가한다.

## 1. 최소 필수 세트

- [x] 로고 심볼 SVG
  - 카드, 말풍선 또는 건네는 동작을 단순하게 표현
  - 밝은 배경과 작은 크기에서도 식별 가능해야 함
- [x] 파비콘 SVG 또는 32×32 PNG
  - 브라우저 탭에서 확인
- [x] 앱 아이콘 512×512 PNG
  - 정사각형 원본
  - 모서리와 핵심 심볼 사이에 안전 여백 확보
- [x] Apple Touch Icon 180×180 PNG
  - 투명 배경보다 완성된 배경색 권장
- [x] OG 공유 이미지 1200×630 PNG 또는 JPG
  - 서비스명 `건넴`
  - 짧은 소개 문구
  - 모바일 카드 화면 또는 로고 포함

## 2. 로고 전달 파일

- [x] 원본 벡터 파일 SVG
- [ ] 밝은 배경용 로고
- [ ] 어두운 배경용 단색 로고
- [x] 심볼만 있는 버전
- [x] `건넴` 워드마크가 포함된 버전
- [ ] 로고 최소 여백과 최소 크기 확인

## 3. 포트폴리오 공개용

- [ ] 프로젝트 대표 이미지 1600×900
- [ ] 포트폴리오 썸네일
- [ ] 모바일 화면 목업 1200×1600 이상
- [ ] 홈 화면 캡처
- [ ] 메인 카드 작성 화면 캡처
- [ ] 수락 또는 거절 페이지 작성 화면 캡처
- [ ] 상대에게 보여주는 카드 화면 캡처
- [ ] 결과 및 답장 카드 화면 캡처
- [ ] 프로젝트 한 줄 소개 문구 확정
- [ ] 사용 기술과 담당 범위 문구 확정

## 4. 디자인 QA용 테스트 에셋

제품에 포함하기 위한 에셋이 아니라 레이아웃 검증에 사용한다.

- [ ] 가로형 사진 1장
- [ ] 세로형 사진 1장
- [ ] 정사각형 사진 1장
- [ ] QR 코드 이미지 1장
- [ ] 투명 배경 PNG 1장
- [ ] 용량이 큰 고해상도 이미지 1장
- [ ] 아주 긴 메인 카드 문구
- [ ] 줄바꿈이 포함된 메인 카드 문구
- [ ] 긴 수락 버튼 문구
- [ ] 긴 거절 버튼 문구
- [ ] 여러 줄 답장 예시
- [ ] 이미지가 전혀 없는 카드 케이스

## 5. 선택 사항: PWA 지원 시

현재 MVP 필수 범위는 아니다.

- [ ] Web App Manifest
- [ ] 192×192 PNG 아이콘
- [ ] 512×512 PNG 아이콘
- [ ] 512×512 마스커블 아이콘
- [ ] PWA 이름과 짧은 이름
- [ ] 테마 색상과 배경 색상
- [ ] 설치 후 시작 URL 및 표시 모드 검증

## 6. 코드 연결 체크리스트

- [x] `public` 또는 정해진 에셋 디렉터리에 파일 배치
- [x] `index.html`에 favicon 연결
- [x] `index.html`에 Apple Touch Icon 연결
- [x] OG title, description, image 메타 태그 연결
- [x] 홈 화면에 로고 적용 여부 결정
- [ ] 모든 이미지에 적절한 대체 텍스트 적용
- [ ] 배포 환경에서 에셋 경로가 정상인지 확인
- [ ] 캐시된 이전 favicon이 아닌 새 아이콘이 표시되는지 확인

## 7. 최종 품질 확인

- [ ] 32px 크기에서도 로고 심볼이 식별됨
- [ ] 아이콘 가장자리가 잘리지 않음
- [ ] 밝은 배경과 어두운 배경에서 대비가 충분함
- [ ] OG 이미지의 핵심 텍스트가 모바일 미리보기에서도 읽힘
- [ ] PNG 이미지가 불필요하게 크지 않도록 압축됨
- [ ] SVG에 불필요한 편집기 정보나 외부 폰트 의존성이 없음
- [ ] 에셋 파일명이 영문 소문자와 하이픈으로 통일됨

## 만들지 않아도 되는 에셋

- [x] 빈 이미지 placeholder는 만들지 않음
- [x] 큰 배경 일러스트는 만들지 않음
- [x] 기능별 장식 아이콘 세트는 만들지 않음
- [x] 수락·거절 캐릭터는 만들지 않음
- [x] 스톡 사진을 기본 콘텐츠로 포함하지 않음
