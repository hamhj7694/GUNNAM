# 건넴 에이전트 운영 체계

이 폴더는 건넴을 기획·설계·구현·검증할 때 사용하는 역할 기반 AI 에이전트 지침을 담는다. 각 에이전트는 독립적으로 판단하되, 공통 문서와 승인 게이트를 따라 협업한다.

## 에이전트 목록

| 역할 | 문서 | 핵심 책임 |
| --- | --- | --- |
| 총괄 지휘자 | `05-lead-orchestrator.md` | 목표 분해, 작업 배정, 충돌 조정, 승인 게이트 관리 |
| 기획자 | `01-product-planner.md` | 사용자 가치, 범위, 정책, 인수 조건 정의 |

## 공통 기준 문서

1. `PRD.md`: 현재 승인된 제품 요구사항
2. `AGENTS.md`: 저장소 작업 절차와 코드 매핑
3. `docs/00-working-rules.md`부터 `docs/05-development-plan.md`: 현재 구현 규칙
4. `docs/06-online-expansion-rfc.md`: 온라인 확장 초안
5. `TODO.md`: 실제 작업 상태의 단일 기준
6. `agents/00-shared-protocol.md`: 에이전트 공통 행동 규칙
7. `agents/ORCHESTRATION.md`: 역할 호출, 산출물 소유권, 회의와 품질 게이트 매핑

RFC와 PRD가 충돌하면 현재 구현에서는 PRD를 따른다. RFC 내용은 사용자가 승인하고 PRD에 반영한 뒤에만 필수 구현 범위가 된다.

## 기본 협업 흐름

```text
사용자 목표
  → 총괄 지휘자: 목표·제약·승인 상태 확인
  → 기획자: 사용자 흐름·정책·인수 조건 정의
  → 설계자: 데이터·권한·시스템 경계 설계
  → 프론트엔드 / 백엔드: 승인된 작업 병렬 구현
  → 품질 관리자: 요구 추적·회귀·보안·완료 판정
  → 총괄 지휘자: 결과 통합·미결정 사항 보고
```

작은 UI 수정처럼 한 역할로 충분한 작업은 모든 에이전트를 거치지 않는다. 총괄 지휘자는 필요한 최소 역할만 활성화한다.

## 승인 게이트

| 게이트 | 통과 조건 | 통과 전 금지 행동 |
| --- | --- | --- |
| G0 문제 정의 | 목표, 사용자, 성공 기준이 명확함 | 구현 착수 |
| G1 제품 승인 | 범위, 정책, 인수 조건이 승인됨 | 제품 동작을 확정하는 코드 변경 |
| G2 설계 승인 | 데이터, API, 권한, 실패 처리가 정의됨 | DB·API·인증 구현 |
| G3 구현 완료 | 코드, 문서, 관련 자동 검증 완료 | 완료 선언 |
| G4 품질 승인 | 핵심 시나리오와 위험 검증 완료 | 배포 준비 완료 선언 |
| G5 배포 승인 | 사용자 또는 권한자의 명시적 승인 | 외부 배포·마이그레이션 실행 |

## 에이전틱 AI 핵심 요소

- 목표 기반 계획: 요청을 검증 가능한 결과로 분해한다.
- 역할 기반 권한: 각 에이전트는 자신의 결정 범위 안에서만 확정한다.
- 상태 기반 실행: `TODO.md`와 승인 게이트로 현재 상태를 추적한다.
- 도구 사용 규율: 읽기·진단은 자율적으로 수행하고, 외부 배포와 파괴적 변경은 승인받는다.
- 증거 기반 완료: 빌드, 테스트, 문서, 인수 조건의 증거 없이 완료로 표시하지 않는다.
- 명시적 인계: 결과물, 결정, 미결정 사항, 위험, 다음 담당자를 구조화해 전달한다.
- 회귀 방지: 현재 사용자 변경과 기존 동작을 먼저 확인하고 사용자 수정 사항을 보존한다.
- 실패 복구: 실패 원인을 기록하고 안전한 대안을 시도하며 같은 실패를 반복하지 않는다.
